The NotAlchemy Engine

This engine was built for CSC 481: Game Engine Foundations. It currently has four implemented games to serve as examples. The supported features of the engine are detailed below.

Features

  • Sprites
    • Images (supports png, jpg, and base64 encoding)
    • Spritestrips
  • Sprite Grouping
  • Sprite Layers
  • Text Display
  • Bounding Boxes
    • Rectangular
    • Circular
  • Collision Detection
    • Point
    • Overlap
  • Tilemaps
  • Level Loading
  • Networking (socket.io)
  • A* Pathfinding

Getting Started

The engine can be downloaded from https://github.ncsu.edu/nmcgove/NotAlchemy/releases. Clone the repository and run npm install to install all dependencies. All engine files are stored under src/engine.

The Game Class

All games extend the Game class. In the constructor of the subclass, all sprites must be constructed and registered to the game with this.spriteLayer.addDrawable(sprite). External content is added using this.addContent(shortName, path), allowing it to be accessed later. The constructor must end with this.loadContent().then(this.start()). The game must also override the loadContent and update methods. Any images used must be added to the image manager with imageManager.addImage(shortName, data[contentName].path.in.json) in loadContent. Anything that needs to be updated every tick must be added to update.

The Sprite Class

All sprites extend the Sprite class. In the constructor of the subclass the following must be set: this.bounds, this.image, and this.image.bounds. The sprite also has to be assigned a size with this.bounds.setSize(w, h) and position with this.bounds.setPosition(x, y) before it will display.
